Публичная версия agent skill для проверки длинных AI-артефактов перед тем, как им доверять, отдавать команде, клиенту или использовать в работе.

Skill полезен, когда AI-агент собрал отчёт, исследование, brief, спецификацию, стратегию, сводку транскриптов или другой документ на основе нескольких источников.

Он не делает модель «всегда правой». Он заставляет модель замедлиться, перечислить допущения, сверить утверждения с источниками и честно сказать, что уже можно использовать, а что ещё нет.

## Какие Ошибки Он Ищет

Quality Check помогает ловить самые опасные сбои в длинной AI-работе:

- факты без источников;
- числа, проценты, даты и периоды, которые модель могла угадать;
- цитаты без понятной атрибуции;
- выводы, которые сильнее, чем позволяют данные;
- скрытые допущения и ограничения;
- уход от исходной задачи;
- уверенное «готово», хотя документ ещё нельзя использовать;
- непроверенный код или обработку данных;
- проблемы source verification в исследовательских документах.

Особенно полезный сценарий: один AI-агент создаёт артефакт, другой проверяет его по тем же локальным источникам.

Например: документ собран в Claude, а проверка запускается через Codex/GPT. Разные семейства моделей часто видят разные слепые зоны.

## Установка

Можно использовать готовый пакет или исходную папку.

Вариант 1: если ваш агент поддерживает `.skill`-файлы, скачайте готовый пакет:

```bash
curl -L -o quality-check.skill \
  https://github.com/MainFrame100/quality-check-skill/raw/main/quality-check.skill
```

Затем импортируйте `quality-check.skill` через механизм импорта вашего agent runtime.

Вариант 2: скопируйте папку skill в директорию skills:

```bash
cp -R quality-check ~/.claude/skills/
```

Если у вашего agent runtime другая структура, используйте его директорию skills.

## Как Вызывать

Автоматическое срабатывание skill не всегда стабильно в разных агентских средах.

Надёжнее вызывать явно:

```text
/quality-check
```

Или использовать понятные ключевые фразы:

```text
Проверь качество ответа.
Проверь качество результата.
Найди ошибки в этом документе.
Проверь факты и источники.
Запусти quality-check для этого артефакта.
Можно ли использовать это as-is?
```

По возможности передавайте:

- сам артефакт;
- исходные файлы или ссылки;
- первоначальную задачу;
- где результат будет использоваться;
- будет ли он отправлен команде, клиенту или опубликован.

## Рекомендуемый Workflow

1. Попросите AI-агента создать артефакт.
2. Сохраните или укажите артефакт и его источники.
3. Запустите `quality-check`.
4. Исправьте все `CONTRADICTS`, неподтверждённые числа, отсутствующие источники и блокирующие `GAP`.
5. Для больших документов проверяйте по разделам.
6. После правок запустите короткую финальную проверку.

## Ограничения

Skill не гарантирует, что найдёт все ошибки.

Он хорошо ловит явные и опасные проблемы: неподтверждённые числа, отсутствующие ссылки, слабое покрытие источниками, уход от задачи и слишком уверенные формулировки.

Он может пропустить более тонкие вещи: слабые гипотезы, слишком широкие обобщения, спорные интерпретации. Это не оракул, а структурированный помощник для проверки.

Если документ большой, не проверяйте всё одним проходом. Делите артефакт на разделы, потому что проверяющая модель тоже может терять детали при слишком большом контексте.

## Как Обновлять

Канонический источник:

```text

```

Если адаптируете skill, сохраняйте поля `metadata.author`, `metadata.source`, `metadata.version` и `license` в `quality-check/SKILL.md`, чтобы было понятно, откуда он взят и где искать обновления.

## Что Считать Хорошим Результатом

Хороший отчёт Quality Check должен отвечать на три вопроса:

1. Можно ли использовать результат как есть?
2. Что конкретно нужно исправить перед использованием?
3. Какие утверждения сверены с источниками, а какие нет?
